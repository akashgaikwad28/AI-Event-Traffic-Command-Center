**GridWise AI** is an open-source, enterprise-grade, real-time AI Traffic Operations Command Center. 

Moving beyond traditional dashboards and ML notebooks, GridWise AI is a complete operational intelligence platform. It ingests live event metadata, runs multi-model inference pipelines to calculate cascading congestion risks, and utilizes Generative AI to recommend dynamic manpower deployments, barricading strategies, and optimal diversion plans.

---

## 🎯 Executive Summary & Problem Solved

GridWise AI forecasts event-related traffic impact and recommends optimal manpower, barricading, and diversion plans using historical and real-time intelligence.

It solves this through the **GORI (GridWise Operational Risk Index)** Engine. By assessing precise geographic coordinates, event severity, rush-hour status, and vehicle counts, the system actively simulates congestion spreads and provides immediate, actionable mitigation plans to city planners, dispatchers, and executive leadership.

---

## ✨ Key Features

- **🔴 Real-Time GORI Scoring**: Computes complex risk tiers (from LOW to CRITICAL) dynamically based on 5 core sub-components: Congestion Risk, Hotspot Severity, Deployment Pressure, Cascading Spread, and Rush-Hour Stress.
- **👮 Resource Optimization Engine**: Calculates the exact number of officers required, ideal deployment zones, and barricade requirements tailored strictly to event severity and available capacity.
- **🗺️ Simulation Engine (What-If Scenarios)**: Proves mitigation impact by modeling traffic spread *before* and *after* GridWise interventions.
- **🤖 Tactical AI Copilot**: Translates raw predictive data into audience-specific narratives (`EXECUTIVE`, `DISPATCHER`, `ANALYST`) using real-time generative AI.
- **📊 Intelligence Reporting**: Generates downloadable PDF/Markdown executive shift handovers and daily operational summaries.
- **⚡ Real-Time Streaming**: High-throughput WebSocket architecture ensuring sub-100ms latency for live alerts and UI updates.

## 🏗️ Architecture & Technology Stack

GridWise AI utilizes a decoupled, high-performance architecture.

### **Backend Stack**
- **Framework**: FastAPI (Python)
- **AI/ML**: Scikit-Learn, XGBoost, Pandas, Numpy, Joblib (Thread-safe Singleton Inferences)
- **Generative AI**: Groq / Gemini via LangChain orchestrators
- **Real-time**: FastAPI WebSockets
- **Concurrency**: Asynchronous event loop (`asyncio`) handling 50+ concurrent inference operations per second gracefully.

### **Frontend Stack**
- **Framework**: React 18, TypeScript, Vite
- **State Management**: Zustand (Multi-store domain architecture)
- **Styling**: TailwindCSS (Dark Enterprise Tactical Theme)
- **Data Visualization**: ECharts, Leaflet (OpenStreetMap)
- **Network**: React Query, native WebSockets

## 🚀 Getting Started

Follow these steps to run the GridWise AI Command Center locally.

### Prerequisites
- Node.js (v18+)
- Python (3.10+)
- Package Managers: `npm` and `pip`

### 1. Backend Setup

1. From the main project directory, create and activate a virtual environment:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```
2. Install dependencies:
   ```bash
   pip install -r requirements/base.txt
   ```
   *(Note: The exact path to requirements may vary depending on project structure, standard is `pip install -r requirements.txt` if at root)*
3. Start the FastAPI server from the project root:
   ```bash
   python -m uvicorn backend.app.main:app --host 0.0.0.0 --port 8000 --reload
   ```

*(The backend will be available at `http://localhost:8000`. API documentation is autogenerated at `/api/v1/docs`.)*

### 2. Frontend Setup

1. Navigate to the frontend directory:
   ```bash
   cd frontend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Start the Vite development server:
   ```bash
   npm run dev
   ```

*(The Command Center will be available at `http://localhost:5173`.)*

## 🧪 Testing & Validation

GridWise AI includes comprehensive testing to ensure production-level thread-safety and sub-second latency across all prediction and AI models.

### 1. Interactive ML Model Testing (Swagger UI)
You can manually test all three of our core AI models (GORI Prediction, Resource Optimization, and Simulation) by injecting your own custom geographical coordinates and incident parameters directly through the browser.

1. Ensure the FastAPI backend is running.
2. Navigate to: **[http://localhost:8000/api/v1/docs](http://localhost:8000/api/v1/docs)**
3. Scroll to the **Predictions**, **Optimization**, and **Simulation** endpoint sections.
4. Click **"Try it out"**, modify parameters, and click **"Execute"** to see live ML output.

### 2. Automated End-to-End Certification
To run the automated Python test suite:
```bash
cd backend
python scripts/run_e2e_tests.py
```
